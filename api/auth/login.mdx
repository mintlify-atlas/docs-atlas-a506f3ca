---
title: 'Login'
description: 'Authenticate a user and obtain an access token'
api: 'POST /api/login'
---

Authenticates a user with their email and password credentials. Upon successful authentication, returns a user object and authentication token that should be used for subsequent API requests.

## Request Body

<ParamField body="email" type="string" required>
  The user's email address used for authentication.
  
  Example: `usuario@empresa.com`
</ParamField>

<ParamField body="password" type="string" required>
  The user's password.
  
  Example: `mySecurePassword123`
</ParamField>

## Response

<ResponseField name="user" type="object">
  Contains the authenticated user's information and authentication token.
  
  <Expandable title="user">
    <ResponseField name="user" type="object">
      The user's profile information.
      
      <Expandable title="user details">
        <ResponseField name="id" type="string">
          Unique identifier for the user.
        </ResponseField>
        
        <ResponseField name="email" type="string">
          The user's email address.
        </ResponseField>
        
        <ResponseField name="firstName" type="string">
          The user's first name.
        </ResponseField>
        
        <ResponseField name="lastName" type="string">
          The user's last name.
        </ResponseField>
      </Expandable>
    </ResponseField>
    
    <ResponseField name="token" type="string">
      JWT authentication token to be used in subsequent API requests. Include this token in the `Authorization` header as `Bearer {token}`.
    </ResponseField>
  </Expandable>
</ResponseField>

## Error Responses

<ResponseField name="error" type="string">
  Error message when authentication fails.
  
  Common error: `Credenciales incorrectas` (Invalid credentials)
</ResponseField>

<RequestExample>

```bash cURL
curl --request POST \
  --url https://api.respondeia.com/api/login \
  --header 'Content-Type: application/json' \
  --data '{
    "email": "usuario@empresa.com",
    "password": "mySecurePassword123"
  }'
```

```javascript JavaScript
const response = await fetch('/api/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    email: 'usuario@empresa.com',
    password: 'mySecurePassword123'
  })
});

const data = await response.json();

if (!response.ok) {
  throw new Error('Credenciales incorrectas');
}

// Store the token for future requests
const { user, token } = data.user;
```

```python Python
import requests

url = "https://api.respondeia.com/api/login"
payload = {
    "email": "usuario@empresa.com",
    "password": "mySecurePassword123"
}
headers = {
    "Content-Type": "application/json"
}

response = requests.post(url, json=payload, headers=headers)

if response.status_code == 200:
    data = response.json()
    user = data['user']['user']
    token = data['user']['token']
    print(f"Logged in as: {user['email']}")
else:
    print("Credenciales incorrectas")
```

## Usage Notes

<Note>
  After successful login, store the authentication token securely. The token should be included in the `Authorization` header for all subsequent API requests:
  
  ```
  Authorization: Bearer {token}
  ```
</Note>

<Warning>
  Never store authentication tokens in plain text or expose them in client-side code repositories. Use secure storage mechanisms appropriate for your platform.
</Warning>
