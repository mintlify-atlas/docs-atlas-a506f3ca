Creates a new user account in the RespondeIA platform. After successful registration, the user can authenticate using the `/api/login` endpoint.

## Request Body

<ParamField body="firstName" type="string" required>
  The user's first name.
  
  Example: `Juan`
</ParamField>

<ParamField body="lastName" type="string" required>
  The user's last name.
  
  Example: `Pérez`
</ParamField>

<ParamField body="email" type="string" required>
  The user's email address. Must be a valid email format and unique in the system.
  
  Example: `usuario@empresa.com`
</ParamField>

<ParamField body="password" type="string" required>
  The user's password. Must meet security requirements (minimum 8 characters recommended).
  
  Example: `mySecurePassword123`
</ParamField>

## Response

<ResponseField name="user" type="object">
  Contains the newly created user's information.
  
  <Expandable title="user">
    <ResponseField name="id" type="string">
      Unique identifier for the newly created user.
    </ResponseField>
    
    <ResponseField name="email" type="string">
      The user's email address.
    </ResponseField>
    
    <ResponseField name="firstName" type="string">
      The user's first name.
    </ResponseField>
    
    <ResponseField name="lastName" type="string">
      The user's last name.
    </ResponseField>
    
    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp of when the account was created.
    </ResponseField>
  </Expandable>
</ResponseField>

## Error Responses

<ResponseField name="error" type="string">
  Error message when registration fails.
</ResponseField>

<ResponseField name="message" type="string">
  Detailed error message describing why registration failed.
</ResponseField>

Common error scenarios:
- Email already registered
- Invalid email format
- Password does not meet security requirements
- Missing required fields

<RequestExample>

```bash cURL
curl --request POST \
  --url https://api.respondeia.com/api/register \
  --header 'Content-Type: application/json' \
  --data '{
    "firstName": "Juan",
    "lastName": "Pérez",
    "email": "usuario@empresa.com",
    "password": "mySecurePassword123"
  }'
```

```javascript JavaScript
const response = await fetch('/api/register', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    firstName: 'Juan',
    lastName: 'Pérez',
    email: 'usuario@empresa.com',
    password: 'mySecurePassword123'
  })
});

const data = await response.json();

if (response.ok) {
  console.log('Account created successfully:', data.user);
  // Redirect to login or automatically authenticate
} else {
  console.error('Registration failed:', data.error);
}
```

```python Python
import requests

url = "https://api.respondeia.com/api/register"
payload = {
    "firstName": "Juan",
    "lastName": "Pérez",
    "email": "usuario@empresa.com",
    "password": "mySecurePassword123"
}
headers = {
    "Content-Type": "application/json"
}

response = requests.post(url, json=payload, headers=headers)

if response.status_code == 200:
    data = response.json()
    user = data['user']
    print(f"Account created for: {user['email']}")
else:
    error = response.json()
    print(f"Registration failed: {error.get('error', 'Unknown error')}")
```

## Usage Notes

</Note>

<Tip>
  Consider implementing email verification after registration to ensure valid email addresses and enhance security.
</Tip>
