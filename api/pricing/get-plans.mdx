## Overview

This endpoint returns a list of all available pricing plans, including their features, pricing information for monthly and annual billing cycles, and display properties.

## Request

<ParamField query="cache" type="string" optional>
  Cache control parameter. Use `no-cache` to bypass cache and fetch fresh data.
</ParamField>

### Headers

No authentication required. This is a public endpoint.

## Response

Returns an array of Plan objects with the following structure:

<ResponseField name="plans" type="Plan[]" required>
  Array of available pricing plans
  
  <Expandable title="Plan object properties">
    <ResponseField name="id" type="string" required>
      Unique identifier for the plan. Possible values: `basico`, `pro`, `agencia`
    </ResponseField>

    <ResponseField name="name" type="string" required>
      Display name of the plan (e.g., "Básico", "Pro", "Agencia")
    </ResponseField>

    <ResponseField name="description" type="string" required>
      Short description of the plan's target audience
    </ResponseField>

    <ResponseField name="priceMonth" type="number" required>
      Monthly price in USD
    </ResponseField>

    <ResponseField name="priceYear" type="number" required>
      Annual price per month in USD (base price for annual calculation)
    </ResponseField>

    <ResponseField name="features" type="string[]" required>
      Array of feature descriptions included in the plan
    </ResponseField>

    <ResponseField name="buttonText" type="string" required>
      Call-to-action text for the plan's button
    </ResponseField>

    <ResponseField name="highlight" type="boolean" required>
      Whether this plan should be highlighted/featured in the UI
    </ResponseField>
  </Expandable>
</ResponseField>

## Integration Example

Here's how the pricing plans are typically used in a React/Next.js application:

```tsx Example Usage
import { useGetPricingPlans } from '@/modules/prices/hooks/useGetPricingPlans';
import { PricingScreen } from '@/modules/prices/screens/pricing-screen';

// ISR Configuration: Revalidate every 24 hours
export const revalidate = 86400;

export default async function PricingPage() {
  // Fetch plans from the API
  const { plans } = await useGetPricingPlans();

  // Pass plans to the pricing screen component
  return <PricingScreen initialPlans={plans} />;
}
```

## Billing Cycle Types

The platform supports two billing cycle types:

<ParamField body="billingCycle" type="string">
  - `monthly` - Monthly billing cycle (uses `priceMonth` value)
  - `annually` - Annual billing cycle (uses `priceYear` value, typically discounted)
</ParamField>

## Caching

This endpoint uses Incremental Static Regeneration (ISR) with a 24-hour revalidation period (`revalidate = 86400` seconds) to ensure optimal performance while keeping pricing data relatively fresh.

## Notes

<Note>
  Currently, the pricing plans are served from a static configuration. In future versions, these may be dynamically loaded from a database or external pricing service.
</Note>

<Tip>
  The `highlight` field is used to mark the recommended or most popular plan in your UI. Typically, only one plan should have `highlight: true`.
</Tip>
